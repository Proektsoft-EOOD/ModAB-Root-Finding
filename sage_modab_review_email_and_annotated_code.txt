Subject: Review comments on SageMath PR #42424 find_root_modab implementation

Dear colleagues,

Thank you for implementing `find_root_modab` in SageMath. I reviewed the proposed implementation against the improved ModAB algorithm described in the paper. The implementation preserves the basic bracketing idea, but there are a few places where the code is not fully faithful to the algorithm and may behave differently after AB fallback.

The most important issue is that the current code uses `y1` and `y2` both as true endpoint values `f(x1), f(x2)` and as Anderson-Björck modified interpolation values. After an AB correction, these values are no longer necessarily the true endpoint function values. If the fallback returns the method to bisection, the next linearity/symmetry test still uses the modified values to compute `ym`, `r`, and `k`, although those quantities are derived for true endpoint values. I recommend keeping true endpoint values (`f1`, `f2`) separate from AB-modified interpolation values (`y1`, `y2`), or at minimum recomputing `y1 = g(x1)` and `y2 = g(x2)` when falling back to bisection.

A second issue is that the switch from bisection to AB is currently applied before the bisection endpoint update. This initializes the fallback threshold using the pre-update width, although the AB phase actually starts after the midpoint has been inserted into the bracket. It also allows the midpoint bisection update to be counted as an AB-side update. The switch should preferably be delayed until after the endpoint update, and the threshold should be initialized from the post-update width.

Additional smaller robustness points are marked in the annotated code: use `isfinite` rather than only `isnan`, count the final residual-check evaluation if reporting `function_calls`, use `<=` in the switching test if matching the paper exactly, and consider returning a guaranteed in-bracket point when terminating by interval width.

I attached an annotated version of the proposed `find_root_modab` implementation. Comments beginning with `# REVIEW:` mark the places that I consider that should be checked or corrected.

Kind Regards,
Latchezar P. Tomov, New Bulgarian University

from math import isnan


def find_root_modab(f, a, b, xtol=10e-13, rtol=2.0**-50, maxiter=100, full_output=False):
    r"""
    Numerically find a root of ``f`` on the closed interval `[a,b]`
    (or `[b,a]`) using the modified Anderson-Bjork bracketing method of
    [GSRTT2026]_.

    The function values at the interval endpoints must have opposite signs,
    unless one endpoint is already a root. Like :func:`find_root`, this
    routine works in fixed (machine) precision.

    Unlike :func:`find_root`, this routine only accepts callables; symbolic
    expressions and symbolic equalities are not dispatched to symbolic root
    finding.

    INPUT:

    - ``f`` -- a function of one variable

    - ``a``, ``b`` -- endpoints of the interval
    - ``xtol``, ``rtol`` -- the routine converges when the enclosing
      interval has width at most ``xtol + rtol*abs(x)``, where ``x`` is the
      current root estimate. Both tolerances must be nonnegative.

    - ``maxiter`` -- integer; if convergence is not achieved in
      ``maxiter`` iterations, an error is raised. Must be nonnegative.

    - ``full_output`` -- boolean (default: ``False``); if ``True``, also
      return an object that contains information about convergence

    EXAMPLES::

        sage: from sage.numerical.optimize import find_root_modab
        sage: find_root_modab(lambda x: x*x - 2, 0, 2) # abs tol 1e-12
        1.414213562373...
        sage: find_root_modab(lambda x: x - 3, 5, -1)
        3.0
        sage: find_root_modab(lambda x: x + 1, -1, 4)
        -1.0

    With ``full_output=True`` a SciPy-style result object is returned::

        sage: root, result = find_root_modab(lambda x: x^3 - 2, 0, 2, full_output=True)
        sage: root # abs tol 1e-12
        1.259921049894...
        sage: result.converged
        True
        sage: result.flag
        'converged'
        sage: result.method
        'modAB'

    The method is a bracketing method, so the endpoints must bracket a
    root::

        sage: find_root_modab(lambda x: x*x + 1, -1, 1)
        Traceback (most recent call last):
        ...
        RuntimeError: f appears to have no zero on the interval

    TESTS::

        sage: find_root_modab(lambda x: x*x - 2, 2, 0) # abs tol 1e-12
        1.414213562373...
        sage: find_root_modab(lambda x: (x - 1)^3, 0, 2)
        1.0
        sage: root, result = find_root_modab(lambda x: x^3 - 2, 0, 2, full_output=True)
        sage: result.function_calls >= 3
        True
        sage: result.iterations >= 1
        True
        sage: from sage.numerical.all import find_root_modab as lazy_modab
        sage: lazy_modab(lambda x: x - 1, 0, 2)
        1.0
        sage: find_root_modab(lambda x: x*x - 2, 0, 2, maxiter=0)
        Traceback (most recent call last):
        ...
        RuntimeError: failed to converge after 0 iterations

    Check that convergence to a discontinuity is rejected, as in
    :func:`find_root`::

        sage: find_root_modab(lambda x: 1/(x - 1) + 1, 0.00001, 2)
        Traceback (most recent call last):
        ...
        NotImplementedError: ModAB failed to find a zero for f on the interval
    """
    if xtol < 0:
        raise ValueError("xtol must be nonnegative")
    if rtol < 0:
        raise ValueError("rtol must be nonnegative")
    if maxiter < 0:
        raise ValueError("maxiter must be nonnegative")
    a = float(a)
    b = float(b)
    if a > b:
        a, b = b, a

    function_calls = 0

    def g(x):
        nonlocal function_calls
        function_calls += 1
        return float(f(x))

    def same_sign(y, z):
        return (y > 0) == (z > 0)

    def result(root, iterations, check=False):
        if check:
            residual = float(f(root))
            # REVIEW:
            # This calls f(root) directly rather than g(root), so this final
            # evaluation is not counted in function_calls. If full_output is
            # meant to report the exact number of evaluations, this should be
            # changed to:
            #
            #     residual = g(root)
            #
            # or function_calls should be incremented explicitly.

            tolerance = max(abs(root * rtol * (initial_y2 - initial_y1)
                                / (b - a)), 1e-6)
            # REVIEW:
            # This residual check is not the same as the documented stopping
            # criterion, which is based on interval width:
            #
            #     x2 - x1 <= xtol + rtol*abs(x)
            #
            # It uses an initial secant-slope scale plus a hard floor of 1e-6.
            # That may be acceptable as a post-check for discontinuities, but
            # it should be documented separately from the main convergence
            # condition. Otherwise users may assume residual convergence is
            # part of the ModAB stopping rule.

            if (isnan(residual)
                or abs(residual) > tolerance):
                raise NotImplementedError("ModAB failed to find a zero for f on the interval")
        if full_output:
            from scipy.optimize import RootResults
            return root, RootResults(root, iterations, function_calls, 0, 'modAB')
        return root

    y1 = g(a)
    if y1 == 0.0:
        return result(a, 0)
    y2 = g(b)
    if y2 == 0.0:
        return result(b, 0)

    if isnan(y1) or isnan(y2) or same_sign(y1, y2):
        # REVIEW:
        # This rejects NaN endpoints, but not +inf or -inf. Infinite function
        # values can later produce invalid secant points, invalid symmetry
        # factors, or NaN in the AB correction. For numerical robustness,
        # prefer:
        #
        #     from math import isfinite
        #     if not isfinite(y1) or not isfinite(y2) or same_sign(y1, y2):
        #
        # and do the same for y3 below.
        raise RuntimeError("f appears to have no zero on the interval")

    initial_y1 = y1
    initial_y2 = y2

    x1 = a
    x2 = b
    bisection = True
    side = 0
    threshold = x2 - x1

    # REVIEW (major design issue):
    # In this implementation the variables y1 and y2 are used for two different
    # purposes:
    #
    #   1. true endpoint values f(x1), f(x2), needed for bracketing,
    #      sign tests, and the bisection-mode symmetry/linearity test;
    #   2. Anderson-Bjork modified endpoint values, needed only for the AB
    #      interpolation formula.
    #
    # After an AB correction, y1 or y2 may no longer equal the true function
    # value at the corresponding endpoint. If the method later falls back to
    # bisection, the bisection-mode test below still computes ym, r, and k from
    # these modified values. That is no longer faithful to the algorithm in the
    # paper, where the linearity/symmetry test is derived from true endpoint
    # values.
    #
    # Suggested structure:
    #
    #     f1, f2 = true endpoint values
    #     y1, y2 = AB-modified interpolation values
    #
    # Use f1/f2 for bracketing and bisection-mode tests; use y1/y2 only for
    # AB secant interpolation. On entering AB mode, set y1, y2 = f1, f2. On
    # fallback, discard the modified values or reset y1, y2 = f1, f2.

    for iteration in range(1, int(maxiter) + 1):
        if bisection:
            x3 = 0.5 * (x1 + x2)
        else:
            x3 = (x1 * y2 - y1 * x2) / (y2 - y1)
            # REVIEW:
            # This formula is fine for the AB interpolation step only if y1 and
            # y2 are intentionally the AB-modified values. It should not use the
            # same variables that later drive bisection-mode diagnostics.

        if x2 - x1 <= xtol + rtol * abs(x3):
            # REVIEW:
            # In AB mode, x3 has just been computed by the secant formula but
            # has not yet been clamped to [x1, x2]. If roundoff places x3 just
            # outside the bracket and the width criterion already holds, this
            # can return an out-of-bracket estimate. Safer options are:
            #
            #   - check the width before computing the AB secant point and
            #     return the midpoint;
            #   - or clamp x3 before using it in the tolerance expression;
            #   - or return 0.5*(x1 + x2) when terminating by interval width.
            return result(x3, iteration, check=True)

        if bisection:
            y3 = g(x3)
            if isnan(y3):
                # REVIEW:
                # Prefer not isfinite(y3), not only isnan(y3).
                raise RuntimeError("f appears to have no zero on the interval")
            ym = 0.5 * (y1 + y2)
            r = 1 - abs(ym / (y2 - y1))
            k = r * r
            # REVIEW:
            # The formulas for ym, r, and k are meaningful only for true
            # endpoint values f(x1), f(x2). If y1 or y2 have been modified by
            # Anderson-Bjork correction during a previous AB phase, then this
            # test no longer measures the actual linearity/symmetry of f on the
            # current interval.
            #
            # This is the main reason to separate true endpoint values (f1/f2)
            # from AB-modified interpolation values (y1/y2).

            if abs(ym - y3) < k * (abs(y3) + abs(ym)):
                # REVIEW:
                # The paper states the switching condition with a non-strict
                # inequality (<=). If exact agreement with the paper is desired,
                # use <= here. This matters in the symmetric boundary case where
                # the equality can hold exactly.

                bisection = False
                threshold = (x2 - x1) * 16
                # REVIEW:
                # The switch to AB is applied before the endpoint update below.
                # However, the AB phase actually begins after this midpoint has
                # been inserted into the bracket. Therefore the fallback width
                # W0 should be the post-update width, not the pre-update width.
                #
                # As written, the threshold is too permissive by about a factor
                # of 2 in this transition, effectively allowing one extra
                # bisection-step lag (C=32 rather than C=16 for the first AB
                # phase).
                #
                # Suggested:
                #     switch_to_ab = True
                #     update the bracket first
                #     then set bisection = False, side = 0,
                #     threshold = (x2 - x1) * 16 using the new width.
        elif x3 <= x1:
            x3, y3 = x1, y1
            threshold *= 0.5
            # REVIEW:
            # Clamping is good for floating-point safety. However, if y1 is an
            # AB-modified interpolation value rather than the true f(x1), then
            # y3 is also not a true function value. This is acceptable only if
            # this branch is treated purely as an interpolation safeguard. It
            # further supports the need to keep true endpoint values separate.
        elif x3 >= x2:
            x3, y3 = x2, y2
            threshold *= 0.5
            # REVIEW:
            # Same issue as above: y2 may be an AB-modified value, not true
            # f(x2).
        else:
            y3 = g(x3)
            if isnan(y3):
                # REVIEW:
                # Prefer not isfinite(y3), not only isnan(y3).
                raise RuntimeError("f appears to have no zero on the interval")
            threshold *= 0.5

        if y3 == 0.0:
            return result(x3, iteration, check=True)

        if same_sign(y1, y3):
            if side == 1:
                m = 1 - y3 / y1
                y2 *= m if m > 0 else 0.5
                # REVIEW:
                # This is the Anderson-Bjork correction and is reasonable in AB
                # mode. However, it changes y2 so that y2 is no longer
                # necessarily equal to the true value f(x2). After fallback,
                # this modified y2 must not be used in the bisection-mode
                # linearity/symmetry test.
            elif not bisection:
                side = 1
                # REVIEW:
                # Because bisection may have been set to False earlier in this
                # same iteration, a midpoint bisection update can be counted as
                # an AB-side update. The switch to AB should be delayed until
                # after the bisection endpoint update; then side is updated only
                # by actual AB iterations.
            x1, y1 = x3, y3
        else:
            if side == -1:
                m = 1 - y3 / y2
                y1 *= m if m > 0 else 0.5
                # REVIEW:
                # Same issue: this modifies y1 for AB interpolation. It should
                # not replace the true endpoint value used for bracketing and
                # switching diagnostics after fallback.
            elif not bisection:
                side = -1
                # REVIEW:
                # Same issue as above: if bisection was set to False earlier in
                # the current midpoint iteration, this bisection step is counted
                # as an AB-side update.
            x2, y2 = x3, y3

        if x2 - x1 > threshold:
            bisection = True
            side = 0
            # REVIEW:
            # This resets the control mode, but it does not undo the
            # Anderson-Bjork modifications applied to y1 or y2.
            #
            # After fallback, the next bisection-mode test computes:
            #
            #     ym = 0.5*(y1 + y2)
            #     r  = 1 - abs(ym/(y2 - y1))
            #
            # Those formulas are derived for true endpoint values f(x1), f(x2),
            # not for AB-modified interpolation values. Therefore the controller
            # is no longer faithful to the paper after the first fallback.
            #
            # Minimal fix:
            #     y1 = g(x1)
            #     y2 = g(x2)
            #
            # Better fix:
            #     maintain f1/f2 as true endpoint values separately from y1/y2,
            #     and reset y1, y2 = f1, f2 when entering or leaving AB mode.

    raise RuntimeError("failed to converge after %s iterations" % maxiter)
